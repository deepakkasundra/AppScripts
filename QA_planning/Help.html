Validate Policy Name

1. Google Sheet names cannot contain more than 100 characters; this option verifies the length of the policy name.

2. It checks if there are duplicate policy names.

3. Reduces errors while creating policy name sheets.

Create Policy Sheets

1. Under the Policy Details menu, select "Create Policy Sheets" after validating the policy name.

2. This creates policy-specific sheets and adds default templates to them.

Generate Hyperlink for Policy Sheets

1. After creating a sheet, go to the Policy Details menu and select "Generate Policy Hyperlink".

2. This generates links for easy access to the respective sheets.

Update Formula

1. Under the Policy Details Menu, select "Update Formula".

2. It dynamically searches for column names and updates the formulas accordingly.

3. After updating, drag the formula down or copy and paste it to the rest of the sheet.

4. You can run this option multiple times to ensure accuracy.

5. Dynamically adds new columns wherever necessary.

Generate Cycle-wise QA Report

1. Under the QA Report Menu, select "Update QA Cycle Wise Report".

2. Once the first QA cycle is complete, click this option to generate the report in the "QA_Cycle_Wise_Report" sheet.

3. The report will be based on the latest data available in the QA_Report sheet.

4. It's recommended to generate cycle-wise QA reports regularly.

Import Executed Data

Note: This option clears the current data in the Policy Name sheet and replaces it with new data from the Execution Sheet.

1. If the execution sheet contains multiple test cases, this option imports them into the respective policy sheets.

2. Go to File > Import > Upload tab > Browse and select the desired file to import.

3. Choose "Insert New Sheet(s)" from the Import Location dropdown and click "Import Data".

4. Rename the newly created sheet to "Execution" and click "Import Executed Data" from the Utility Menu.

5. Check the import status in the Execution sheet's last column.

Consolidate Data

The "Consolidate Data" Option available under "QA Report". Which helps consolidates data from multiple sheets dynamically. It merges data from each policy sheet into a single “Consolidated Data”.

Key Features

  • Dynamic Header Mapping: Maps headers from a template sheet ("Test Cases Format") and ignores blanks or headers labeled "Go to Main Page."
  • Automatic Sheet Handling: Creates "Consolidated Data" if missing, or prompts the user to clear it if it exists.
  • Column Mapping & Data Extraction: Matches headers from each policy sheet with the template. Missing headers are flagged as “Header not found.”
  • Data Consolidation: Appends each policy's data into "Consolidated Data," prefixed by the policy name.
  • Conditional Formatting: Highlights cells labeled “Header not found” with a light red background, refreshing any existing formats.

Usage Instructions:

  1. Ensure "QA_Report" and "Test Cases Format" sheets are present as they provide policy names and header templates.
  2. If prompted to clear "Consolidated Data," verify its contents and proceed.

This feature enhances data reporting, analysis, and validation processes.
